PRD vs Implementation

The original PRD specified slightly different hex values than what was implemented in the Tailwind config. Both are documented here for reference.

ColorPRD SpecImplementationDifference
Fern #4A7C59 #4A6741 → Darker, less saturated
Sage #8FAF8B #A3B18A → Lighter, warmer
Terracotta #C97B5A #C4704B → Slightly deeper
Blush #E8B4A0 #E8C4B8 → Cooler, pinker
Warm Brown #7A5C44 #6B4D3E → Darker
Oat #F2EDE4 #F0EBE3 → Negligible
Which to use? Implementation values (right column) are the source of truth for all digital work. PRD values may still appear in early documents. Both are correct for their context — when in doubt, use the Tailwind config values.

CSS Variables Reference

Custom properties defined in the site's globals.css. These are the computed runtime values.

/* globals.css — root custom properties */
--background: #FAF7F2; /* Page background (cream) */
--foreground: #2C2C2C; /* Default text (charcoal) */
--linen-wash: rgba(250,247,242,0.65); /* Semi-transparent overlay */
--focus-ring: #5B7F5E; /* WCAG-compliant focus outline */
